System Architecture

A distributed, cloud-native system running on AWS ECS with intelligent orchestration

Specter operates as a fully autonomous, cloud-native system running on AWS infrastructure. An Orchestrator service continuously monitors JIRA for new tickets, automatically determines which agent should handle each request, and coordinates the entire workflow from detection to delivery. Five specialized agent workers‚ÄîDARCI, SCRIBE, MERLIN, HERMES, and SENTRY‚Äîrun independently as ECS Fargate containers, each powered by the Phoenix autonomous engine and Claude Agent SDK.

The workflow is elegantly simple: tickets labeled with an agent name trigger automatic processing. Agents claim their assigned tasks, analyze the relevant code repositories from a shared EFS volume, and for BEACON, integrate with external APIs like MOANA for SQL generation. Results are uploaded to S3, and the Orchestrator posts these results back to JIRA as formatted comments. The entire cycle‚Äîfrom ticket creation to solution delivery‚Äîcompletes in under 8 minutes, with configuration lookups finishing in under a minute. All data is encrypted and HIPAA compliant, with horizontal scalability built in through stateless worker design.
